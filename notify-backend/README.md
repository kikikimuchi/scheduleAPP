# メール通知バックエンド（無料・Google Apps Script）

アプリを閉じていても、各タスクの「指定分前」にメールで通知します。
**Firebaseの課金（Blaze）は不要**。Google Apps Script の無料枠だけで動きます。

## 仕組み
1. アプリが「今後7日分の予定」を Firestore の `schedule_notify/upcoming` に公開する。
2. このスクリプトが **1分ごと** に動き、その予定を読む。
3. 各タスクの `時刻 − 指定分` になったら **Gmailでメール送信**。

```
[アプリ] --(予定を書き出し)--> [Firestore] <--(1分毎に読む)-- [Apps Script] --(メール)--> [あなた]
```

## セットアップ（5分・1回だけ）

### 1) アプリ側で通知をONにする
1. アプリ → 設定（⚙）→「🔔 メール通知（5分前）」
2. **「メール通知を有効にする」にチェック**
3. **通知先メールアドレス**を入力（例: あなたのGmail）
4. **何分前**に通知するか（初期値 5）
5. 「通知設定を保存」をタップ
   → これで Firestore に予定が公開されます。

### 2) Apps Script を作る
1. https://script.google.com を開く →「新しいプロジェクト」
2. 既定の `Code.gs` の中身を全部消し、本フォルダの **`Code.gs` を貼り付け**て保存
   - `PROJECT_ID` / `API_KEY` はこのアプリ用に設定済みなので変更不要です。

### 3) 通知のスイッチを入れる（トリガー作成）
1. 上部の関数選択で **`setupTrigger`** を選び「実行」
2. 初回は認可ダイアログが出る → アカウントを選び、「詳細」→「(安全でないページに)移動」→ 許可
   （Gmail送信と外部接続の権限を承認します）
3. 「1分ごとのトリガーを作成しました」と出れば完了。

### 4) テスト（任意）
- 関数 **`sendTestMail`** を実行 → 設定したメールにテストが届けばOK。
- 実通知のテストは、アプリでタスク時刻を「今 + 指定分」に設定して数分待つと届きます。

### 5) iPhoneで受け取る
- 受信に使うメール（Gmail等）アプリの**通知をON**にしておく。
  これでアプリを閉じていてもメール通知が届きます。

## 注意・制限
- **無料Gmailの送信上限は1日100通**まで（タスク数×日でこれを超えないよう注意）。
- タスクごとに1通届きます。多すぎる場合は通知する分を増やす/タスクを整理してください。
- アプリを**一度も開かない日**は予定が更新されません（公開は最大7日先まで先読みするので、
  普段使いで毎日開いていれば問題ありません）。
- 時刻の基準は日本時間（`Asia/Tokyo`）。`Code.gs` の `TZ` で変更可能。
- 予定の公開には Firestore の読み取りが**公開（ルールで read 許可）**である必要があります。
  既存アプリがログイン不要で動いているので、通常はそのままで動作します。

## 仕組みを止めたいとき
- アプリ設定で「メール通知を有効にする」のチェックを外して保存（送信されなくなる）。
- 完全に止めるなら Apps Script の「トリガー」画面で `checkAndNotify` のトリガーを削除。
